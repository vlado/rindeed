api_search(what, where, options={})

Sends search query to Indeed XML interface for what (job title, keywords or company name) in the where (city, state or zip code) area and returns results as Rindeed::Results instance (Array with Indeed metadata)

Options

  • :sort - Sort by relevance or date
  • :start - Start results at this result number.
  • :limit - Maximum number of results returned per query.
  • :fromage - Number of days back to search. Max is 30.
  • :filter - Filter duplicate results. 0 turns off duplicate job filtering.
  • :latlong - If latlong=1, returns latitude and longitude information for each job result.
  • :radius - Show jobs within (radius) miles radius
  • :sr - Source employer (directhire) or job site (recruiters)
  • :jt - Filter by job type (fulltime|parttime|contract|internship|temporary)
  • :salary - Desired salary. :salary => 90000 will show only jobs with salary greater then $90000

There can be even more options. See www.indeed.com/jsp/xmlsample.jsp or www.indeed.com/advanced_search for more info

Metadata

  • uri - Full uri used to fetch results
  • success - Request was succesfull true or unsuccesfull false
  • query - what
  • location - where
  • dupefilter - Duplicates are filtered true or not false (see options[:filter])
  • highlight - ?
  • totalresults - Total results for provided query
  • start - see options[:start]
  • end - options[:start] + options[:limit]

paginate(what, where, options={})

Similar to api_search (find) method, with differnce that it returns Rindeed::Collection allowing us to use will_paginate functionality for pagination

Options

  • :page - REQUIRED, but defaults to 1 if false or nil

All other options are identical as with api_search method

See mislav.uniqpath.com/static/will_paginate/doc/ for more info
